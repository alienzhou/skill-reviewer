# D01: Checklist 条目设计

> Status: ✅ Confirmed | Date: 2026-02-28

## Context

基于 Agent Skills 构建指南（PDF），设计 Skill Reviewer 的 Checklist 检查条目。

## Decision

采用 20 条检查项，分为 4 个类别：

### 📁 Structure（结构）- 4 项

| ID | 检查项 | 评判标准 | 检测方式 |
|----|--------|----------|----------|
| S1 | **SKILL.md 存在** | ✅ 文件存在且命名正确（大小写敏感）<br>❌ 文件不存在或命名变体 | 程序 |
| S2 | **文件夹命名** | ✅ kebab-case<br>⚠️ 包含下划线<br>❌ 包含空格或大写 | 程序 |
| S3 | **目录结构** | ✅ 合理使用可选目录<br>⚠️ 缺少但不影响功能<br>❌ 包含禁止的 README.md | 程序 |
| S4 | **名称一致性** | ✅ 文件夹名与 name 字段一致<br>⚠️ 不一致但可工作 | 程序 |

### 📋 Format（格式）- 5 项

| ID | 检查项 | 评判标准 | 检测方式 |
|----|--------|----------|----------|
| F1 | **YAML 分隔符** | ✅ 正确使用 `---`<br>❌ 缺少或格式错误 | 程序 |
| F2 | **name 字段** | ✅ 存在、kebab-case、无空格/大写<br>⚠️ 小问题<br>❌ 缺失或错误 | 程序 |
| F3 | **description 字段** | ✅ 存在且 <1024 字符<br>❌ 缺失或超长 | 程序 |
| F4 | **无禁止内容** | ✅ 无 XML 标签 `<>`<br>❌ 包含禁止内容 | 程序 |
| F5 | **可选字段格式** | ✅ license/metadata 格式正确<br>⚠️ 有但格式不标准 | 程序 |

### 📝 Content（内容）- 8 项

| ID | 检查项 | 评判标准 | 检测方式 |
|----|--------|----------|----------|
| C1 | **Description WHAT** | ✅ 清晰说明用途<br>⚠️ 模糊<br>❌ 缺失 | 模型 |
| C2 | **Description WHEN** | ✅ 包含触发条件<br>⚠️ 不够具体<br>❌ 无触发描述 | 模型 |
| C3 | **指令可执行** | ✅ 具体步骤<br>⚠️ 可更具体<br>❌ 模糊笼统 | 模型 |
| C4 | **错误处理** | ✅ 有 Troubleshooting<br>⚠️ 简单提及<br>❌ 完全没有 | 模型 |
| C5 | **示例** | ✅ 有完整示例<br>⚠️ 示例不完整<br>❌ 无示例 | 模型 |
| C6 | **引用链接** | ✅ 链接正确<br>⚠️ 部分失效<br>❌ 错误或缺失 | 模型 |
| C7 | **渐进披露** | ✅ SKILL.md 聚焦核心<br>⚠️ 偏长但可接受<br>❌ 过于臃肿 | 模型 |
| C8 | **关键指令突出** | ✅ 使用标记，置顶<br>⚠️ 结构尚可<br>❌ 被埋没 | 模型 |

### 🎯 Trigger（触发设计）- 3 项

| ID | 检查项 | 评判标准 | 检测方式 |
|----|--------|----------|----------|
| T1 | **正向触发** | ✅ 具体短语<br>⚠️ 不够丰富<br>❌ 无触发短语 | 模型 |
| T2 | **触发范围** | ✅ 范围适当<br>⚠️ 稍宽或稍窄<br>❌ 误触发或难触发 | 模型 |
| T3 | **负向触发** | ✅ 明确排除场景<br>⚠️ 无但范围尚可<br>N/A 简单 Skill 不需要 | 模型 |

## Rationale

1. **分组方式**：按检测类型分组（Structure → Format → Content → Trigger），因为我们做事后 Review，不是开发过程指导
2. **检测分层**：程序检测 9 项（S1-S4 + F1-F5），模型检测 11 项（C1-C8 + T1-T3）
3. **评判标准**：将好实践/坏实践融入各检查项的 ✅⚠️❌ 三档标准中
